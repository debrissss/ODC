# 📚 MaCVi 2026 热红外目标检测改进点详细说明与消融实验指南 (Beginner Friendly)

本文档旨在针对 **MaCVi 2026 热红外目标检测挑战赛** 提供一份“保姆级”的模型优化解析。不论您之前是否接触过目标检测或者 MMDetection 框架，阅读本指南后您都能清晰掌握我们的优化动机、方案原理以及如何开展严谨的“消融实验”（Ablation Study）。

---

## 🧐 什么是消融实验 (Ablation Study)？
在深度学习算法优化中，我们经常会同时引入多个改进点（例如：改了数据增强、又换了模型主干、还调了损失函数）。如果直接测试最终模型，分数虽然提高了，但我们**无法得知究竟是哪一个改进点起到了决定性作用**。
消融实验的思路就像控制变量法：我们在基线网络 (`Baseline`) 的基础上，**每次只加入其中一个改进点**，观察指标的变化。这有助于科研汇报和论文撰写，也是深刻理解网络瓶颈的最佳实践。

基于此理念，我们将先前提出的 4 项提效方案做了**完全代码解耦**，全部存放在 `configs/faster_rcnn/` 路径下以便独立运行：

## 🛠️ 改进点深度解析与对应消融脚本

### 1️⃣ 消融核心 1：数据增强与多尺度训练优化
> **对应脚本**：`configs/faster_rcnn/ablation_1_data_aug.py`

- **针对痛点**：验证集结果显示小目标（`mAP_s`）精度低下。原始基线固定把图片缩放为 `(800, 600)` 的小尺寸，容易把原本只有几个像素的海上远距小船直接缩没（失真）。另外热红外图中缺少纹理，模型极其容易对有限的训练集背景造成过拟合。
- **改进原理**：
  1. **采用多尺度缩放 (Multi-scale Resize)**：在配置文件中指定 `scale=[(1333, 600), (1333, 960)]`。在每次训练输入前，系统将在给定宽高范围内随机选取一个尺度对图片进行放大。这有助于维持小目标的清晰度。
  2. **红外领域特定增强 (Thermal Noise Injection)**：编写了 `custom_modules/transforms.py`。规则上我们禁止任意色彩反转以保护物理场温度的极性归一化结果，所以只在图像上随机撒上符合原图方差的“高斯微噪点”（Gaussian Noise）。模型将不再死记硬背像素分布，学习能力更强。

### 2️⃣ 消融核心 2：网络架构与特征融合改进
> **对应脚本**：`configs/faster_rcnn/ablation_2_pafpn.py`

- **针对痛点**：原始基线使用的是基础的特征金字塔（`FPN`）。对于严重存在“同类物体尺度跨度极大”（近处的船非常大，远处的船极小）的航海数据集，原本自顶向下的单向融合信息流不够强大，底层位置信息流失会导致定位不准。
- **改进原理**：由于在短时间内直接更换如 Cascade R-CNN 的全套组件需要极大配置依赖（我们在 MMDetection 的轻量级实施原则下做最有效的平替），我们将金字塔升级为 **PAFPN (Path Aggregation Network)**。
  - **PAFPN 在常规 FPN 基础中，增加了一根“从下往上”的信息传递路径**。因此低层高分辨率的精准定位信息（对小目标极其重要）能够迅速传递给负责大基准框预测的深层网络，从而提升所有尺度的检测精度。

### 3️⃣ 消融核心 3：小目标专属微调 (Anchor & Loss)
> **对应脚本**：`configs/faster_rcnn/ablation_3_small_object.py`

- **针对痛点**：默认的候选框发生器 (Anchor Generator) 尺寸设为 `scales=[8]`。尺寸 8 远大于图片最远端的部分船只像素，导致从一开始候选框就圈不到目标。此外，默认使用的 L1 距离损失对那些有偏移但不相交的检测框缺乏优化方向。
- **改进原理**：
  1. 将 Anchor `scales` 放缩扩展到了 `[2, 4, 8]`，同时把长宽比 `ratios` 拓展到 `[0.5, 1.0, 2.0, 3.0]`。比值 3.0 专门应对船只由于各种透视角度形成的修长横截面。
  2. 引入 **GIoU Loss (Generalized IoU)**。不同于 L1 距离，GIoU 计算了预测框与真实框的交并比，甚至在两框不重合时也能通过“包含二者最小外接矩形”进行惩罚梯度计算，促使边界框向目标稳准狠地靠拢。

### 4️⃣ 消融核心 4：训练策略调整
> **对应脚本**：`configs/faster_rcnn/ablation_4_schedule_3x.py`

- **针对痛点**：原基线的 12 Epoch 会导致模型对这批全新特征或多尺度图还没“看熟”就终止了。
- **改进原理**：继承并使用了基于新建 `configs/_base_/schedules/schedule_3x.py`。将训练拉长到 **36 个 Epoch**，配合第 24 和 33 Epoch 触发的标准 0.1 倍衰减，极大降低了损失波动，避免欠拟合。

## 🏆 终极综合版模型 (Final Comprehensive Version)
> **对应脚本**：`configs/faster_rcnn/faster_rcnn_r50_pafpn_3x_thermal_final.py`

这是把上述**四大类改进强强联合**的最优版本。当您完成消融实验后，可以使用它去冲击排行榜。它集合了 PAFPN 架构、小目标适配尺寸、多尺度抗锯齿缩放与极致训练时长度，是一套从输入到输出全面强化的战斗机模型。

## 🖥️ 运行实验的正确姿势 (How to run)
打开终端窗口，按照规则约束（不进行硬代码插入参数覆盖，一切靠启动脚本驱动）：

**运行单独改进项验证 (如验证数据增强)**：
```bash
python tools/train.py configs/faster_rcnn/ablation_1_data_aug.py
```

**运行大一统终极版本获取最好结果**：
```bash
python tools/train.py configs/faster_rcnn/faster_rcnn_r50_pafpn_3x_thermal_final.py
```
