# 📊 实验与结果分析

- **配置**：`configs/faster_rcnn/faster_rcnn_r50_fpn_1x_ms_cocopretrained.py`
- **架构**：Faster R-CNN + ResNet-50 + FPN
- **训练策略**：1x Schedule (12 Epochs)

## 第二轮实验：综合优化版 (Full Optimization)
- **配置**：`configs/faster_rcnn/faster_rcnn_r50_pafpn_3x_thermal_final.py`
- **架构**：Faster R-CNN + ResNet-50 + **PAFPN**
- **训练策略**：**3x Schedule (36 Epochs)**
- **数据增强**：**多尺度 [(1333,600)~(1333,960)]** + RandomFlip + **ThermalNoiseInjection**
- **损失函数**：RoI Head 使用 **GIoU Loss** 替代 L1 Loss
- **Anchor**：scales=[2,4,8], ratios=[0.5,1.0,2.0,3.0]

| 指标 | 数值 |
|------|:----:|
| **mAP** | 0.3510 |
| **mAP_50** | 0.6820 |
| **mAP_75** | 0.3090 |
| **mAP_s** | 0.2780 |
| **mAP_m** | 0.3670 |
| **mAP_l** | 0.5030 |

## 对比分析

| 指标 | 基线 | 优化后 | 变化 | 变化幅度 |
|------|:----:|:----:|:----:|:----:|
| **mAP** | 0.3400 | 0.3510 | **+0.011** | +3.2% |
| **mAP_50** | 0.6770 | 0.6820 | **+0.005** | +0.7% |
| **mAP_75** | 0.2830 | 0.3090 | **+0.026** | +9.2% |
| **mAP_s** | 0.2610 | 0.2780 | **+0.017** | +6.5% |
| **mAP_m** | 0.3850 | 0.3670 | **-0.018** | -4.7% |
| **mAP_l** | 0.5470 | 0.5030 | **-0.044** | -8.0% |

### ✅ 正面提升
1. **mAP 总体提升 +3.2%**：核心排名指标由 0.34 → 0.351，方向正确。
2. **mAP_75 提升显著 (+9.2%)**：GIoU Loss 的效果最为突出——高 IoU 阈值下的定位精度大幅增强，模型对边界框的回归更加精准。
3. **mAP_s 小目标提升 +6.5%**：多尺度训练 + 更小的 Anchor scales 确实发挥了作用，小目标的召回和精度均有改善。

### ⚠️ 存在问题
1. **mAP_m 下降 -4.7%，mAP_l 下降 -8.0%**：中、大目标精度出现了显著回退。可能原因：
   - Anchor scales 增加了极小尺度 `2` 和 ratio `3.0` 后，候选框密度大增，对中大目标引入了更多假阳性干扰。
   - PAFPN 自下而上的路径在强化底层特征的同时，可能对高层大目标语义信息产生了稀释效应。
   - 训练从 12 拉到 36 Epoch，在数据量较小（训练集仅 ~2000 标注）的情况下可能出现了对小尺度特征的过拟合。
2. **总体提升幅度有限**：mAP 仅提升了 1.1 个点，说明在 Faster R-CNN + ResNet-50 的框架天花板下，单靠调参和局部改动很难取得质的飞跃。
