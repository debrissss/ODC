# 🚀 组会汇报：MaCVi 2026 热红外目标检测挑战赛 方案优化总结

## 1. 🎯 挑战与基线痛点分析
在本次 CVPR 2026 MaCVi 热红外目标检测挑战赛中，我们首先在一阶段基线 Faster R-CNN (ResNet-50) 模型上获得了验证集上的初步成绩：
> **`mAP`: 0.3400**, **`mAP_50`: 0.6770**

然而深入剖析指标发现核心瓶颈：
- **`mAP_s` (小目标精度) 仅为 0.2610**
- 热红外图像缺乏特征纹理，传统自然光场景下的增强与架构未能充分挖掘有效上下文。

## 2. 🛡️ 架构合规与规范执行 (依从 Code Guide)
借助于 MMDetection 和 MMEngine 框架设计理念的纯粹性，本次实施**严格秉承 Agent 编码规范**：
- **注册表驱动**：放弃随意修改框架源码流，所有新增功能（如定制 Transform）通过 `@TRANSFORMS.register_module()` 动态桥接。
- **无头全链路管控**：完全阻断任何渲染/展示类阻断型 API，后续所有推断只采取挂载探针落盘机制和 Config-as-code 原理运作于远程 Linux 队列。
- **物理与领域约束**：绝不在增强通道插入颜色翻转等破坏热极性归化的转换函数。

## 3. 🛠️ 本次提效落地实施详情 (Implementation)

### 3.1 自定义领域增强模块 (`custom_modules/transforms.py`)
由于热红外场景下单调且缺乏边界信息，基于 `BaseTransform` 抽象写出了专用增强方法 `ThermalNoiseInjection`：
- 在不破坏热能分布极性的前提下截断地注入高斯噪声，强化网络对细小目标的感受野识别，迫使模型学习更为稳健的热力分布而不仅是对比度极值。

### 3.2 自适应训练调优 (Schedule 3x)
新建 `configs/_base_/schedules/schedule_3x.py`：
- 由于数据经过多尺度放缩且网络加深，将 Epoch 极大地从 12 次拉升至 **36 次**。
- 设置更为平滑的 MultiStepLR 步骤衰减以防止梯度激荡。

### 3.3 大一统配置文件集成 (`faster_rcnn_r50_fpn_3x_thermal_improved.py`)
在保持对基础库配置继承的同时，进行了核心模块手术：
1. **多尺度训练 (Multi-scale Resize)**：`scale=[(1333, 600), (1333, 960)]` 取代基础单尺度，有效抑制目标下采样中的失真现象，直击 `mAP_s` 的弱点。
2. **Anchor 微调**：`scales=[2, 4, 8]` 添加更小像素块生成单元以契合极远帆船/障碍物。
3. **GIoU Loss 更替**：将原本粗放的 L1 回归更换为惩罚并集计算的 GIoU，收敛更快预测更贴合热源中心。

## 4. 📈 预期收益与下一步建议计划
通过以上“数据多尺度 + 小目标 Anchor 设计 + 物理学安全增强”的三管齐下操作：
- 预计小目标精度 (`mAP_s`) 提升 3% - 5% 及以上。
- 更长的 schedule 将带来全局泛化性能。
- **下一步推进方案**：验证以上配置能正常在显存内消化后，即刻准备执行 `tools/train.py` 命令。成功收敛后可快速在 `test.py` 输出预测 JSON 打包提交挑战赛官方。
